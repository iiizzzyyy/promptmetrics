## Notes

- **Webhook routes** (`/webhooks/github`) intentionally do not use `requireScope`. They are not API-key-authenticated; they validate via `X-Hub-Signature-256` HMAC against `GITHUB_WEBHOOK_SECRET`. Scope semantics do not apply to GitHub webhook callers, who are infrastructure rather than tenant principals. Per CLAUDE.md pitfall #6, this route also uses Express's raw body parser; do not add JSON middleware in front of it.
- **API-key routes** require `admin` scope (not `write`) because key issuance/revocation is a privileged tenant-management operation.
- **Audit-log read** requires `admin` scope despite being a GET because audit data is sensitive.
- **Health endpoints** are intentionally public (no auth) per CLAUDE.md pitfall #7. `/health/deep` is handled directly by the raw `http.Server` in `server.ts` and is not subject to Express middleware.
- **Playground POST routes** call third-party LLM providers and emit audit log entries; they mutate billable state and are correctly gated by `write`.
